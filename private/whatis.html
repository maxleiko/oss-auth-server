OneShotSoftware framework

This framework proposes the foundation for creating a moving target defense architecture for distributed software. This framework enables distributed to change their behavior, the component interaction, the graphical interface and their architecture configurations dynamically while preserving the requirements and integrity of services. The main goal of the oneshotsoftware framework is to hinder the adversaryâ€™s capabilities in scanning or discovering software vulnerabilities targets, launching attacks and creating botnets structure.

OneShotSoftware architecture

The framework architecture is based on the following basis:

  • Kevoree for the architectural model and deployment management.
  • Spoon for the source to source transformation engine.

Based on these two foundations, we are building a set of model for capturing components interaction, components interface, components graphical interface and we are building a set of libraries to create disruption,processor to automatically inject countermeasures withing the component source code. Next we use Kevoree and Docker to automatically deploy the components variant for each new user.
